🔄 Computed Fields — Fields that Calculate Themselves

Jab field ki value DB mein stored nahi hoti, balki automatically calculate hoti hai based on other fields

📖 What is a Computed Field?

Computed field wo hota hai jiska value store nahi hota (ya store bhi ho sakta hai) — balki @api.depends ke through calculate hota hai. Jaise total = price × quantity.

💡 Key Rule: Computed field ki value manually set nahi kar sakte. Sirf compute method se calculate hoti hai. store=True karo toh DB mein save hoga.

Computed Field Types

📐
Basic Computed Field
compute parameter se define karo
@api.depends
▼
🗣️ Hinglish: Field mein compute='method_name' likho. Har baar dependencies change hone pe method call hoga.
Python
1234567
total = fields.Float(compute='_compute_total')

@api.depends('price', 'quantity')
def _compute_total(self):
    for rec in self:
        rec.total = rec.price * rec.quantity
🎯
@api.depends (Dependencies)
Batata hai ki konse fields change pe recalculate karna hai
DECORATOR
▼
🗣️ Hinglish: @api.depends('field1', 'field2'). Dot notation bhi chalega — 'partner_id.country_id.name'.
Python
1234567
@api.depends('line_ids.subtotal')
def _compute_amount(self):
    for rec in self:
        rec.total_amount = sum(rec.line_ids.mapped('subtotal'))
💾
store=True vs False
store=True = DB mein save karo, store=False = on-the-fly calculate
OPTION
▼
Python
123
total = fields.Float(compute='_compute_total', store=True)
# store=True → search, filter, group kar sakte ho
# store=False (default) → har baar calculate karo
↩️
inverse() — Write Computed Fields
Jab user computed field manually set kare
ADVANCED
▼
🗣️ Hinglish: Rare scenario — user computed field mein value daale (like manual override). Toh inverse method define karo jo uska value wapas source fields mein distribute kare.
Python
123456789
total = fields.Float(compute='_compute_total', inverse='_inverse_total')

def _inverse_total(self):
    for rec in self:
        rec.price = rec.total / rec.quantity  # Reverse calculate
💡 Inverse rarely used. Mostly for compatibility with external APIs.
🔍
search() — Searchable Computed
Computed field pe search karne allow karo
ADVANCED
▼
Python
123456789
is_vip = fields.Boolean(compute='_compute_is_vip', search='_search_is_vip')

def _search_is_vip(self, operator, value):
    # Return domain for search
    return [('total_invoiced', >, 100000)]

Related Fields (computed shortcut)
Simply related='source.field' — automatic computed
SHORTCUT
▼
🗣️ Hinglish: Agar tujhe kisi Many2one ke field ko yahan bhi dikhana hai toh related attribute use kar. Odoo apne aap @api.depends aur compute handle kar lega.
Python
12345
partner_country = fields.Char(related='partner_id.country_id.name')
# No compute method needed!
# Agar partner ka country badla toh yeh auto-update hoga

📝 Computed Cheat Sheet

Cheat Sheet
123456
total = fields.Float(compute='_compute')
@api.depends('price', 'qty')
def _compute(self):
    for rec in self: rec.total = rec.price * rec.qty

# store=True → save in DB
# inverse → write support
# search → search support
# related → auto computed field
