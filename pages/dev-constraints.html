🚫 Constraints — Data Validation on Save

Record save hone se pehle check karo ki data valid hai ya nahi — galat data DB mein nahi jaane doge

📖 What are Constraints?

Constraints wo rules hain jo Odoo save se pehle check karta hai. Agar rule fail hua toh record save nahi hoga aur error dikhega. Do types hain:

  • Python Constraints — @api.constrains('field') — complex logic, cross-field checks
  • SQL Constraints — _sql_constraints — fast, database level (unique, check)
💡 Diff: Python constraints = record save se pehle run hote hain. SQL constraints = database level pe hamesha enforce.

@api.constrains — Python Constraint
Complex validation: price negative nahi ho sakta
@api.constrains
▼
🗣️ Hinglish: Jab record save ho, @api.constrains('field') check karta hai. Agar validation fail hogi toh ValidationError raise karo — save nahi hoga.
Python
12345678910
from odoo.exceptions import ValidationError

@api.constrains('price')
def _check_price(self):
    for rec in self:
        if rec.price < 0:
            raise ValidationError(
                "Price negative nahi ho sakta bhai! 😤"
            )
🏢 Real-life: Koi bhi product -₹100 price ke saath save nahi ho sakta. Ya sale order mein discount -20% nahi daal sakta. Ye sab constraints se check karo.
🗄️
_sql_constraints — Database Level
Fast, DB level validation — unique, check
SQL
▼
🗣️ Hinglish: Database level constraints bahut fast hote hain. Python validation se pehle hi DB reject kar deta hai. _sql_constraints mein list of tuples: (name, sql_expression, message).
Python
123456789
_sql_constraints = [
    ('positive_price',
     'CHECK(price >= 0)',
     'Price 0 se kam nahi ho sakta bro!'),
    ('unique_product_code',
     'UNIQUE(code)',
     'Yeh code already exist karta hai!'),
    ('end_after_start',
     'CHECK(date_end >= date_start)',
     'End date start date se pehle nahi ho sakti'),
]
💡 SQL vs Python: SQL constraints DB level pe hain — bypass nahi ho sakte. Python constraints only through ORM (can be bypassed with raw SQL). Best practice: dono use karo.

Date Validation — End >= Start
Sabse common problem: date validation
PYTHON
▼
🗣️ Hinglish: "End date, start date se pehle kyun daal raha hai bhai?" — yeh Odoo mein sabse common validation hai. Dono Python aur SQL mein kar sakte ho.
Python
12345678
@api.constrains('date_start', 'date_end')
def _check_dates(self):
    for rec in self:
        if rec.date_end and rec.date_start:
            if rec.date_end < rec.date_start:
                raise ValidationError(
                    "End date start se pehle nahi ho sakti! 🗓️"
                )

Conditional Constraint — Complex Logic
Field A aur Field B ka combination unique hona chahiye
PYTHON
▼
🗣️ Hinglish: Kabhi kabhi complex condition hoti hai jo SQL mein nahi likhi ja sakti. Tab Python constraint use karo — jaise "ek user ka ek hi draft sale order ho sakta hai same partner ke liye".
Python
12345678910111213
@api.constrains('partner_id', 'state')
def _check_duplicate_draft(self):
    for rec in self:
        if rec.state == 'draft':
            duplicates = self.search([
                ('partner_id', =, rec.partner_id.id),
                ('state', =, 'draft'),
                ('id', !=, rec.id),
            ])
            if duplicates:
                raise ValidationError(
                    f"Customer {rec.partner_id.name} ka already ek draft order hai! 📋"
                )
