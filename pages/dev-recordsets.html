📦 Recordsets — The Odoo Data Container

Odoo mein har data piece ek recordset hai — samjho isse toh ORM master ban jaoge

📖 What is a Recordset?

Odoo mein jab bhi tum koi record access karte ho — search(), browse(), ya self — toh tumhe ek recordset milta hai. Yeh Python ka ordered set hai jismein 0, 1, ya multiple records ho sakte hain. Think of it as a special list that can hold database records.

┌─────────────────────────────────────────────────────┐ │ RECORDSET TYPES │ ├─────────────────────────────────────────────────────┤ │ │ │ Empty: self.env['res.partner'] │ │ → res.partner() # 0 records │ │ │ │ Singleton: self.env['res.partner'].browse(42) │ │ → res.partner(42,) # 1 record │ │ │ │ Multi: self.env['res.partner'].search([]) │ │ → res.partner(1,2,3,42) # N records │ │ │ │ Truth: empty = False, non-empty = True │ │ Length: len(recordset) │ │ Access: recordset[0], recordset[-1] │ │ Slice: recordset[:5] │ └─────────────────────────────────────────────────────┘
💡 Key Rule: Recordset HAMESHA ek specific model ka hota hai. res.partner ka recordset mein product.product nahi daal sakte.

Recordset Basics & Truthiness
Empty = False, Non-empty = True, len() works
CORE
▼
🗣️ Hinglish: Recordset Python mein truthy/falsy hai. Empty recordset = False, kuch bhi hai toh True. if partner: se check kar sakte ho ki record hai ya nahi.
Python
12345678910111213141516171819
# ── Creating recordsets ──
empty     = self.env['res.partner']           # Empty
single    = self.env['res.partner'].browse(1)  # One record
multiple  = self.env['res.partner'].search([]) # Many records

# ── Truthiness ──
print(bool(empty))    # → False
print(bool(single))   # → True
print(bool(multiple)) # → True

# ── Common pattern ──
partner = order.partner_id
if partner:
    print(f"Customer: {partner.name}")
else:
    print("No customer set!")

# ── Length ──
print(len(multiple))  # → 150 (number of records)

Iteration (for loop)
Har record pe ek ek karke loop chala sakte ho
ITERATE
▼
🗣️ Hinglish: for rec in self: — yeh Odoo ka bread and butter hai. Har method mein multi-record support ke liye loop lagao. Har iteration mein rec ek singleton recordset hoga.
Python
123456789101112131415
# ── Standard pattern ──
partners = self.env['res.partner'].search([], limit=5)
for partner in partners:
    print(f"{partner.id}: {partner.name}")
    # partner is a singleton here (1 record)

# ── Computed field — always loop over self ──
def _compute_total(self):
    for rec in self:
        rec.total = rec.price * rec.qty

# ── With enumeration ──
for i, rec in enumerate(self, 1):
    print(f"{i}. {rec.name}")
# 1. Rahul  2. Priya  3. ABC Corp
⚠️ NEVER assume self is singleton! Unless you call self.ensure_one(). Button handlers mein self ek record hota hai, but computed/write mein multiple ho sakte hain.

Field Access (dot notation)
Singleton pe direct dot access — multi pe loop karo
READ
▼
🗣️ Hinglish: Singleton pe partner.name se seedha value milti hai. Multi-record pe direct access karo toh error aayega — loop ya mapped() use karo.
Python
123456789101112131415
# ── Singleton — direct access ──
partner = self.env['res.partner'].browse(42)
print(partner.name)    # → 'Rahul Sharma'
print(partner.email)   # → 'rahul@x.com'

# ── Many2one → returns recordset ──
print(partner.country_id)       # → res.country(105,)
print(partner.country_id.name)  # → 'India'
print(partner.country_id.code)  # → 'IN'

# ── One2many → returns multi recordset ──
print(partner.child_ids)        # → res.partner(5,6,7)

# ── Multi-record access → ERROR! ──
# partners.name → ValueError (which name? ambiguous!)

Python list jaisa — index aur slice karo
RECORDSET
▼
Python
12345678910
partners = self.env['res.partner'].search([], limit=10)

first  = partners[0]      # First record (singleton)

Operations

filtered, mapped, sorted — the power trio

🔽
.filtered(func)
Python mein filter — naya recordset return karta hai
RECORDSET
▼
🗣️ Hinglish: Search ke baad Python level pe aur filter lagana ho — lambda do ya field name do. Naya filtered recordset milega.
Python
12345678910111213141516
partners = self.env['res.partner'].search([])

# ── Lambda filter ──
with_email = partners.filtered(
    lambda p: p.email and '@gmail' in p.email
)

# ── String shortcut (boolean fields) ──
companies = partners.filtered('is_company')

# ── Complex condition ──
vip = partners.filtered(
    lambda p: p.total_invoiced > 100000
              and p.country_id.code == 'IN'
)
print(f"VIP customers: {len(vip)}")
💡 search() vs filtered():
• search() → SQL (fast, large data)
• filtered() → Python (slow, but can access computed fields/methods)
🗺️
.mapped(func_or_field)
Har record se value extract — list ya recordset return
RECORDSET
▼
🗣️ Hinglish: mapped('name') = sab records ke names ki list. mapped('country_id') = unique countries ka recordset. Lambda bhi de sakte ho.
Python
123456789101112131415161718
# ── Extract field values → list ──
names = partners.mapped('name')
# → ['Rahul', 'Priya', 'ABC Corp']

# ── Dot notation → nested values ──
countries = partners.mapped('country_id.name')
# → ['India', 'USA', 'India']

# ── Relational field → recordset (unique!) ──
all_countries = partners.mapped('country_id')
# → res.country(105, 233)  ← only unique records

# ── Lambda → list ──
formatted = partners.mapped(
    lambda p: f"{p.name} ({p.email or 'N/A'})"
)

# ── Get all product IDs from order lines ──
product_ids = order.order_line.mapped('product_id.id')
💡 String vs Lambda return:
• String with relational field → recordset (deduplicated)
• String with regular field → list
• Lambda → always list
📊
.sorted(key, reverse)
Recordset ko Python mein sort karta hai
RECORDSET
▼
Python
1234567891011
# ── Sort by field name ──
by_name = partners.sorted(key='name')
by_name_desc = partners.sorted(key='name', reverse=True)

# ── Sort by lambda ──
by_revenue = partners.sorted(
    key=lambda p: p.total_invoiced,
    reverse=True
)

# ── Default sort (no key) = by _order attribute ──
natural = partners.sorted()
💡 search(order='name') = DB sort (fast). .sorted('name') = Python sort (can use computed fields).

Set Operations

Union, intersection, subtract, membership

➕
set_a | set_b (Union)
Merge — duplicates auto-remove
SET OP
▼
Python
123456789
mumbai = Partner.search([('city', =, 'Mumbai')])
delhi  = Partner.search([('city', =, 'Delhi')])

both = mumbai | delhi
# All partners from Mumbai OR Delhi (no duplicates)

# ── Collect partners from multiple orders ──
all_customers = self.env['res.partner']
for order in orders:
    all_customers |= order.partner_id

Properties & Safety

IDs, ensure_one, exists, concatenation

🔢
.ids / .id
Record IDs as Python integers
PROPERTY
▼
Python

partner = Partner.browse(42)
print(partner.id)     # → 42 (int)

partners = Partner.search([], limit=5)
print(partners.ids)   # → [1, 2, 5, 8, 42] (list)

# ── Use in domains ──
domain = [('partner_id', 'in', partners.ids)]

☝️
.ensure_one()
Exactly 1 record assert — safety check
SAFETY
▼
Python
123456
def action_confirm(self):
    self.ensure_one()  # Crash if 0 or 2+ records
    # Safe to use self.field_name now
    self.state = 'confirmed'
    self.date_confirmed = fields.Datetime.now()
    self._send_email()
✅
.exists()
Dead records filter out — sirf existing return kare
SAFETY
▼
Python
1234567
# ── Filter out deleted records ──
recs = Partner.browse([1, 2, 99999])
valid = recs.exists()  # → only [1, 2]

# ── Safe pattern ──
if record.exists():
    record.do_something()
🔗
set_a + set_b (Concatenation)
Join without dedup — duplicates ALLOWED
SET OP
▼
🗣️ Hinglish: | duplicates remove karta hai, + nahi karta. Jab tujhe same record multiple baar chahiye (rare) tab + use kar.
Python
1234567
a = Partner.browse([1, 2])
b = Partner.browse([2, 3])

union  = a | b   # → (1, 2, 3)    deduped
concat = a + b   # → (1, 2, 2, 3) with duplicate

# Usually | is what you want

Relational Navigation

Many2one, One2many, Many2many traverse

🔗
Relational Field Navigation
Dot notation se related records traverse karo
NAVIGATION
▼
🗣️ Hinglish: Odoo ki power yeh hai ki dot notation se related models mein jump kar sakte ho. order.partner_id.country_id.name — 3 models traverse ek line mein!
Python
1234567891011121314151617181920212223
order = self.env['sale.order'].browse(1)

# ── Many2one → singleton recordset ──
customer     = order.partner_id           # res.partner(42,)
country      = order.partner_id.country_id # res.country(105,)
country_name = order.partner_id.country_id.name  # 'India'

# ── One2many → multi recordset ──
lines = order.order_line  # sale.order.line(1,2,3)
for line in lines:
    print(f"{line.product_id.name}: {line.price_subtotal}")

# ── Many2many → multi recordset ──
tags = partner.category_id  # res.partner.category(1,5,8)

# ── Chain mapped for deep navigation ──
all_products = order.order_line.mapped('product_id')
# → product.product(10, 20, 30)

# ── Get all customer countries from multiple orders ──
orders = Order.search([('state', =, 'sale')])
countries = orders.mapped('partner_id.country_id')
# → Unique countries of all confirmed order customers
🏢 Real-life: Sale order se invoice tak: order.invoice_ids.mapped('payment_ids') — ek line mein order → invoices → payments traverse!

Common Mistakes & Fixes
Yeh galtiyan mat karna — time bachega
IMPORTANT
▼
Python
1234567891011121314151617181920212223242526272829303132333435
# ❌ MISTAKE 1: Accessing field on multi-record
partners = Partner.search([])
name = partners.name  # → ERROR! Which partner's name?
# ✅ FIX: Loop or mapped
names = partners.mapped('name')

# ❌ MISTAKE 2: Comparing with None/False
if partner.country_id == None:  # WRONG
    ...
# ✅ FIX: Use truthiness
if not partner.country_id:
    print("No country set")

# ❌ MISTAKE 3: Forgetting loop in compute
def _compute_total(self):
    self.total = self.qty * self.price  # ERROR if multi!
# ✅ FIX: Always loop
def _compute_total(self):
    for rec in self:
        rec.total = rec.qty * rec.price

# ❌ MISTAKE 4: Comparing recordset with int
if partner.id == partner:  # WRONG (int vs recordset)
    ...
# ✅ FIX: Compare properly
if partner.id == 42:       # int vs int ✓
if partner == other_partner: # recordset vs recordset ✓

# ❌ MISTAKE 5: N+1 query problem
for p in partners:
    print(p.country_id.name)  # Each triggers a query!
# ✅ FIX: Prefetch or use mapped
data = partners.mapped(lambda p: (p.name, p.country_id.name))
# Or use search_fetch() in Odoo 17+

📝 Recordset Cheat Sheet

Cheat Sheet
1234567891011121314151617181920212223
# ═══ BASICS ═══
bool(rs)          # Truthy check
len(rs)           # Count
rs[0]  rs[-1]     # Index → singleton
rs[:5]  rs[2:5]   # Slice → recordset
rec in rs         # Membership check

# ═══ POWER TRIO ═══
rs.filtered(func)  # Filter → recordset
rs.mapped(func)    # Map → list or recordset
rs.sorted(key)     # Sort → recordset

# ═══ SET OPS ═══
a | b             # Union (no dupes)
a & b             # Intersection
a - b             # Subtract
a + b             # Concat (with dupes)

# ═══ SAFETY ═══
rs.ensure_one()   # Assert exactly 1
rs.exists()       # Filter dead records
rs.id             # Single ID (int)
rs.ids            # All IDs (list)
