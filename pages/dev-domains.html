Problem 1: "AND vs OR confusion"

🛑 Problem: Mujhe aise customers chahiye jo India ke hain YA (gmail use karte hain AND active hain). Lekin domain galat lag raha hai.
✅ Solution: Prefix notation use karo — '|' OR ke liye, '&' AND ke liye (default AND hota hai).
Python
1234567891011121314151617
# Requirement: (Country = India) OR (Email has gmail AND Active = True)

# ❌ Wrong - This means: India OR (gmail AND active)
# But default AND hota hai, so yeh banega:
# (India AND gmail) OR active - Galat!

# ✅ Correct - Explicit OR at start
domain = [
    '|',   # ← First OR operator
    ('country_id.code', =, 'IN'),
    '&',   # ← AND for next two
    ('email', 'ilike', 'gmail'),
    ('active', =, True),
]

# Visual: ['|', A, '&', B, C]
# Tree:        OR
#            /  \
#           A   AND
#               / \
#              B   C

Problem 2: "Many2one field blank hai tab domain fail"

🛑 Problem: Partner select nahi kiya toh partner_id.city access karte waqt error aata hai.
✅ Solution: Pehle check karo ki field set hai ya nahi. parent.field safe access ke liye.
Python
1234567891011
# ❌ Error if partner_id is empty
# domain = [('partner_id.city', '=', 'Mumbai')]

# ✅ Safe way
domain = [
    ('partner_id', '!=', False),  # Pehle ensure karo set hai
    ('partner_id.city', =, 'Mumbai'),
]

# Ya phir onchange mein handle karo:
@api.onchange('partner_id')
def _onchange_partner(self):
    if self.partner_id:
        return {'domain': {'child_ids': [('city', =, self.partner_id.city)]}}
    else:
        return {'domain': {'child_ids': [('id', =, False)]}}

Problem 3: "Dynamic domain based on user — Security issue"

🛑 Problem: Sales user ko sirf apne assigned customers dikhane chahiye. Lekin domain XML mein hardcoded hai.
✅ Solution: ir.actions.act_window mein dynamic domain lagao ya search method override karo.
XML + Python
12345678910111213141516171819
# ── Method 1: Action mein user_id filter ──
# XML:
<record id="action_my_customers" model="ir.actions.act_window">
    <field name="name">My Customers</field>
    <field name="res_model">res.partner</field>
    <field name="domain">[('user_id', '=', uid)]</field>
</record>

# ── Method 2: Override search (Record Rules jaisa) ──
def search(self, args, offset=0, limit=None, order=None, count=False):
    # Sales user hai toh filter lagao
    if self.env.user.has_group('sales_team.group_sale_salesman'):
        if not self.env.user.has_group('sales_team.group_sale_manager'):
            args += [('user_id', =, self.env.uid)]
    return super().search(args, offset, limit, order, count)

Problem 4: "Date range filter — Last 30 days"

🛑 Problem: Dashboard mein sirf last 30 days ke orders dikhane hain.
Python
123456789101112131415
from datetime import datetime, timedelta

# ── Last 30 days ──
thirty_days_ago = datetime.now() - timedelta(days=30)

domain = [
    ('date_order', >=, thirty_days_ago),
    ('state', =, 'sale'),
]

# ── Current Month ──
start_of_month = datetime.now().replace(day=1)

# ── This Financial Year ──
# Odoo mein usually env.company.compute_fiscalyear_dates() use karte hain
start, end = self.env.company.compute_fiscalyear_dates(datetime.now())
domain = [('date', '>=', start), ('date', '<=', end)]

Problem 5: "Related field pe domain lagana hai"

🛑 Problem: Product select karte waqt sirf uss category ke products dikhane hain jo partner ke allowed categories mein hain.
✅ Solution: domain attribute in XML with parent reference ya onchange mein dynamic domain return karo.
XML + Python
123456789101112131415
# ── XML: Parent field reference ──
<field name="product_id" 
       domain="[('categ_id', '=', parent.allowed_categ_id)]"/>

# ── Python: Dynamic based on context ──
allowed_products = self.env.context.get('allowed_product_ids', [])
domain = [('id', 'in', allowed_products)]

# ── Complex: Partner ke hisaab se product filter ──
@api.onchange('partner_id')
def _onchange_partner_product_domain(self):
    if self.partner_id:
        allowed_categs = self.partner_id.allowed_category_ids.ids
        return {
            'domain': {'product_id': [('categ_id', 'in', allowed_categs)]}
        }
    return {'domain': {'product_id': [('id', =, False)]}}

Problem 6: "Hierarchy — Sirf iske children dikhane hain"

🛑 Problem: Manager ne apne neeche ke employees dikhane hain (self + all subordinates recursively).
✅ Solution: child_of operator — yeh recursive search karta hai.
Python
12345678910111213
# ── child_of: Self + All descendants ──
domain = [('employee_id', 'child_of', self.env.user.employee_id.id)]

# ── parent_of: Self + All ancestors ──
domain = [('id', 'parent_of', some_employee_id)]

# ── Example: Manager ka team ──
class HrLeave(models.Model):
    # ...
    
    def action_approve(self):
        # Sirf apne team ke leaves approve kar sakta hai
        if self.employee_id not in self.env['hr.employee'].search(
            [('id', 'child_of', self.env.user.employee_id.id)]
        ):
            raise AccessError("Not your team member!")

Quick Reference: Common Patterns

Patterns
12345678910111213
# Empty/Set check
[('field', '=', False)]        # Empty
[('field', '!=', False)]       # Set (any value)

# List containment
[('id', 'in', [1,2,3])]        # In list
[('id', 'not in', [4,5])]      # Not in list

# Text search
[('name', 'ilike', 'raju')]   # Case insensitive
[('code', '=like', 'INV-%')]   # Starts with INV-

# Date ranges
[('date', '>=', start), ('date', '<=', end)]

# Many2one exists
[('partner_id', '!=', False)]  # Has partner
