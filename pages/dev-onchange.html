⚡ Onchange — UI Events that React Instantly

Jab user form mein kuch change kare, toh turant kuch aur change ho jaye

📖 What is Onchange?

Onchange ek Python decorator hai jo form view mein user ke field change karne PE call hota hai. Ye sirf UI mein hota hai — jab save nahi hua hai. Code ya API se call nahi hota.

💡 Very Important Difference: Onchange = Form UI event (unsaved). Computed = Always (saved/unsaved both). Dono alag hain.

@api.onchange('partner_id')
def _onchange_partner(self):
    if self.partner_id:
        self.phone = self.partner_id.phone
        self.email = self.partner_id.email
        self.city  = self.partner_id.city
    else:
        self.phone = False
        self.email = False
        self.city  = False

@api.onchange('quantity')
def _onchange_qty(self):
    # ── Warning message ──
    if self.quantity > 1000:
        return {
            'warning': {
                'title': 'Large Quantity!',
                'message': '1000+ units — sure hai?'
            }
        }

# ── Dynamic domain on Many2one field ──
@api.onchange('category_id')
def _onchange_category(self):
    return {
        'domain': {
            'product_id': [('categ_id', =, self.category_id.id)]
        }
    }
# Now product dropdown only shows selected category products

Onchange vs Computed Fields
Kab kya use kare? — Important Decision
DECISION
▼
Python
12345678910111213141516171819202122
# ═══ Rule of Thumb ═══
#
# Use COMPUTED when:
# - Value hamesha consistent honi chahiye
# - Form save, import, API, code — sab jagah same
# - Value depend on other fields (total = price * qty)
# - store=True for search/filter
#
# Use ONCHANGE when:
# - Sirf FORM UI mein react karna hai
# - User ko warning message dikhana hai
# - Many2one ka domain change karna hai
# - Data set karna hai jo override ho sakta hai
#
# Example: Product price → tax calculation
#   onchange: set default tax when product changes
#   computed: actually calculate tax amount
# Dono ek saath use ho sakte hain!
