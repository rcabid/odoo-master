🔧 ORM — Every Method Explained

Har ek ORM method — real-life examples, syntax, aur Hinglish mein samjhaya

📖 What is the ORM?

Odoo ka ORM Python classes ko PostgreSQL tables mein convert karta hai. Sirf Python likho — SQL ki zarurat nahi. Har model = table, har field = column.

💡 Rule: Odoo mein sab data operations — create, read, update, delete — ORM se hote hain. Direct SQL avoid karo.

def _name_search(self, name='', domain=None,
                  operator='ilike', limit=100, order=None):
    domain = domain or []
    if name:
        domain = ['|',
            ('name', operator, name),
            ('phone', operator, name),
        ] + domain
    return self._search(domain, limit=limit, order=order)

API Decorators

@api.onchange('field')
UI mein field change pe turant action
DECORATOR
▼
🗣️ Hinglish: Customer select kiya → address auto-fill. Product select kiya → price aaya. Yeh sirf UI event hai — DB mein kuch nahi hota save se pehle.
Python
1234567891011121314
@api.onchange('partner_id')
def _onchange_partner(self):
    if self.partner_id:
        self.phone = self.partner_id.phone
        self.email = self.partner_id.email

# ── Warning message ──
@api.onchange('quantity')
def _onchange_qty(self):
    if self.quantity > 1000:
        return {'warning': {
            'title': 'Bahut zyada!',
            'message': '1000+ sure hai?'
        }}
💡 onchange = sirf UI. computed = hamesha (code/import/API sab).
🔄
@api.depends('field')
Computed field ki dependencies define karta hai
DECORATOR
▼
🗣️ Hinglish: total = price × qty. Jab price ya qty badle toh total bhi badle — @api.depends yeh batata hai ki KAB recalculate karna hai.
Python
123456789101112
total = fields.Float(compute='_compute_total', store=True)

@api.depends('price', 'quantity', 'discount')
def _compute_total(self):
    for rec in self:
        rec.total = rec.price * rec.quantity * (1 - rec.discount/100)

# ── Depends on One2many lines ──
@api.depends('line_ids.subtotal')
def _compute_amount(self):
    for rec in self:
        rec.total_amount = sum(rec.line_ids.mapped('subtotal'))
⚠️ store=True → DB mein save, searchable. store=False → on-the-fly, not searchable.
